## What this does NOT establish (attribution value ≈ zero)

This is the load-bearing honest finding for the Wei Dai hypothesis:

- **The key was made with GnuPG, using GnuPG defaults.** The cipher/hash/zip preference lists are the canonical GnuPG default vectors. They carry **no individuating signature** — millions of keys look identical. They do *not* point to Crypto++, to a Mac, or to any particular person.
- **This directly undercuts the original doc's "Phase 1: match Crypto++ code signatures in Cicada's PGP tooling."** PGP keys are generated by GnuPG, not by an author's own C++. There is no Crypto++ fingerprint to find in a GnuPG-produced key. The premise was a category error.
- **A 4096-bit RSA key + AES-256 preference describes a population in the millions** (every privacy-aware GnuPG user who bumped the key size). As discriminating evidence for "Wei Dai specifically," its likelihood ratio is ~1. It moves the needle essentially not at all.
- **Timestamps are operator-settable.** Even the creation date is only weak evidence; it's consistent, but it is not tamper-proof attribution.
