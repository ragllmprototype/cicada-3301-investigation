# Cicada 3301 — The Wei Dai Hypothesis: A Falsification-First Forensic Investigation

**Investigator:** Cooldash · **Analysis:** Claude Code
**Date:** 2026-06-12
**Verdict:** Hypothesis **not supported**, and *mildly disconfirmed* by the first genuinely discriminating evidence. Posterior moved **~2% → ~1%.**

---

## 1. The claim under test

> **H1:** Wei Dai (b-money author, Crypto++ maintainer, Satoshi correspondent) architected Cicada 3301 as a post-Satoshi continuation of the cypherpunk recruitment mission.

An earlier draft argued *for* H1 and reported 30–95% confidence. This investigation re-ran the question under a **falsification-first** rule: authenticate every artifact, assign each fact a likelihood ratio, run controls, and try to *break* the hypothesis rather than decorate it.

## 2. Method

1. **Authenticate before analyze.** Nothing enters analysis unless it carries a valid signature from the cryptographic root of trust.
2. **Likelihood ratios, not vibes.** A fact only moves confidence if it is more expected under H1 than under H0. Facts equally expected either way (LR≈1) are discarded as evidence.
3. **Controls.** Every test applied to Wei Dai is applied identically to other pool members (Adam Back, Nick Szabo, Hal Finney).
4. **Real-person discipline.** Wei Dai is a living, private individual; conclusions stay at "hypothesis," never assertion.

## 3. What we built

- **Root of trust:** genuine 3301 key `6D854CD7 9333 22A6 01C3 286D 181F 01E5 7A35 090F`, RSA-4096, **created 2012-01-05 03:39:43 UTC**, UID `Cicada 3301 (845145127)`.
- **Authenticated corpus:** **21** PGP-signed messages spanning 2012-01-05 → 2017-04-04, each verified GOOD against the genuine key (`docs/04`).

## 4. Findings

### F1 — A hoax was caught on the very first artifact
The top-Googling GitHub `PGPKey.txt` is **not** the real key: real fingerprint `CEBB2647…7B4ADD02`, RSA-3072, **created and revoked in 2016**, Gmail UID. Its trick: the UID *display name* was set to the literal string `7A35090F` (the real key's short ID) to fool anyone grepping for it. **Lesson:** short IDs/display names are attacker-controlled; only full-fingerprint matching authenticates. Had we skipped Phase 4, the whole investigation could have analyzed a troll's key.

### F2 — One archived message was corrupted; we recovered the authentic one
The `2013-01-opening-book-code` copy in a popular archive failed verification (BADSIG). Cause: a **dropped opening line** (`"Welcome again."`) broke the signature hash. An authentic, GOOD-verifying copy was recovered from a second archive (signed 2013-01-03 04:33 UTC). Demonstrates why signature-checking beats trusting any single transcription.

### F3 — PGP forensics has authentication value but ZERO attribution value
The genuine key is **stock GnuPG output** — default AES-256/SHA-256/ZLIB preference vectors, bumped to 4096-bit. It identifies *no one*. Critically, this **refutes the original plan** to "match Crypto++ code signatures in Cicada's PGP tooling": PGP keys are generated by GnuPG, not by an author's C++, so there is no Crypto++ fingerprint to find. The premise was a category error. (`docs/02`)

### F4 — Auditing the original evidence: almost everything is LR ≈ 1
| Original "evidence" | Verdict |
|---|---|
| Cypherpunk ideology of 3301 | non-discriminating (true of whole pool) |
| Wei Dai's crypto expertise | non-discriminating (true by construction) |
| "Employment gap / ghost period" | **base-rate trap** — private engineers default to no footprint |
| Launch 8.5 months after Satoshi exit | post-hoc window — many fit |
| RSA-4096 / "Mac opsec" | stock tooling / speculation — millions fit |

Multiplying an honest prior (~0.02) by a chain of 1's leaves the posterior at ~0.02. The original's high confidence confused **consistency** ("could have") with **evidence** ("would have"). (`docs/03`)

### F5 — The first real discriminating evidence points AWAY from Wei Dai
The authenticated corpus yields exactly two markers with discriminating potential (`docs/05`, `docs/06`):

| Marker | CICADA | Wei Dai (authenticated prose) | Match? |
|---|---|---|---|
| **Spelling** | **British** (`organisations`) | **American** (`-ize`, 0 British in 1,358 words) | **No — contradicts** |
| **Spacing** | **100% double** (13/13 + recovered) | **0% double** (0/1,358) | **No — contradicts** |

Controls: Szabo and Finney = American. **Adam Back** = British (`optimised`) — the only pool member whose spelling matches Cicada. So the lone discriminating signal is **exclusionary against Wei Dai** and, if anything, leans toward the pool's British member — though "British spelling" is shared by ~2 billion people and could be deliberate obfuscation, so it is **not** an identification of Back.

## 5. Honesty ledger (limits)

- **Tiny corpus:** ~480 prose words, heavy duplication, only **one** British-vs-American opportunity (n=1) and 13 spacing instances. These can *nudge*, never settle.
- **Temporal gap:** Wei Dai's comparison text (b-money) is from 1998; habits drift (though spelling-nationality rarely flips).
- **Marker ≠ identity:** British spelling excludes US-native natural writers; it does not name anyone.
- **No Tier-1 evidence exists** (no testimony, no leaked logs). We did not pretend otherwise.

## 6. Verdict

| | Original draft | This investigation |
|---|---|---|
| P(Wei Dai is a 3301 architect) | 30–95% | **~1%** |
| Evidence quality | "strong, coherent" | consistent but non-discriminating; first real test is *disconfirming* |
| "Crypto++ code match" plan | central next step | **refuted** (category error) |

**Wei Dai cannot be ruled out** — the prior is small but nonzero, and the markers are thin. But the disciplined result is unambiguous in direction: when the hypothesis finally met data that could distinguish him from the field, it went **down**, not up. Cicada's verified voice is British-spelled and double-spaced; Wei Dai's authenticated voice is neither.

This is the intended outcome of a falsification-first method: a cherished hypothesis, honestly tested, surviving only as a low-probability possibility rather than a confident claim.

## 7. What would change the verdict

- **Toward H1:** a larger authenticated 3301 prose corpus showing American spelling + single-spacing; any Tier-1 link (testimony, logs); infrastructure metadata tying Wei Dai to genuine 3301 servers.
- **Against H1 (further):** authenticated Wei Dai writing from ~2012 confirming consistent American/single-spaced habits; positive stylometric match of Cicada prose to a *different* identified author.
